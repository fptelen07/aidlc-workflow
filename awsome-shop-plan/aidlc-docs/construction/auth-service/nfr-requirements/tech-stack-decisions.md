# Unit 2: auth-service — 技术栈决策

---

## 1. 后端框架

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 后端框架 | 待定 — 用户将提供统一技术框架 | 所有微服务使用统一框架 |
| 架构分层 | 待定 — 用户将提供分层规范 | Controller → Service → Repository |

> 技术框架和架构分层将在实现阶段由用户提供，设计阶段保持技术无关。

---

## 2. 安全组件

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 密码加密 | bcrypt (cost factor = 10) | 业界标准，抗彩虹表攻击 |
| 令牌方案 | JWT (HS256) | 无状态认证，API 网关校验 |
| 令牌过期 | 24 小时（可配置） | 环境变量 JWT_EXPIRATION |
| 退出策略 | 前端清除令牌 | MVP 最简方案，不维护黑名单 |

---

## 3. 数据访问

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 数据库 | MySQL 8.4（共享 Unit 7 基础设施） | auth_db 独立 database |
| 字符集 | utf8mb4 | 支持中文姓名 |
| 连接方式 | 通过 Docker DNS 连接 mysql:3306 | 环境变量配置 |

---

## 4. 跨服务通信

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 通信协议 | HTTP REST | 服务间同步调用 |
| 调用目标 | points-service 内部接口 | POST /api/internal/points/init |
| 超时时间 | 3 秒 | 失败降级，不影响注册 |
| 服务发现 | Docker DNS | http://points-service:8003 |

---

## 5. 性能目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| P95 响应时间（一般接口） | ≤ 200ms | 查询类接口 |
| P95 响应时间（注册/登录） | ≤ 500ms | 含 bcrypt 操作 |
| 并发处理 | 不做特殊优化 | MVP 阶段用户量小 |
| 分页上限 | 100 条/页 | 防止大查询 |

---

## 6. 依赖关系

| 依赖 | 类型 | 说明 |
|------|------|------|
| MySQL (auth_db) | 数据存储 | Unit 7 提供 |
| points-service | 跨服务调用 | 注册时初始化积分（可降级） |
| JWT_SECRET | 共享配置 | 与 api-gateway 共享 |
