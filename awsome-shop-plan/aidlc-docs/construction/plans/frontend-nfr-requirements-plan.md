# Unit 1: awsomeshop-frontend — NFR 需求计划

---

## 概述
本计划用于收集前端应用的非功能性需求，包括性能、可访问性、浏览器兼容性、安全性等方面的决策。

---

## 设计问题

### Q1: 浏览器兼容性要求
前端需要支持哪些浏览器？

- **A**: 仅支持现代浏览器（Chrome/Firefox/Safari/Edge 最新2个主版本）
- **B**: 支持 IE11 及以上（需要 polyfill）
- **C**: 仅支持 Chrome（内部系统，可限定浏览器）

[Answer]: A

---

### Q2: 首屏加载性能目标
首屏加载时间（FCP）的性能目标是什么？

- **A**: P95 ≤ 2秒（标准 Web 应用）
- **B**: P95 ≤ 3秒（可接受的内部系统）
- **C**: 不设具体目标（MVP 阶段不关注）

[Answer]: A

---

### Q3: JavaScript 包体积限制
是否需要限制 JavaScript 包的体积？

- **A**: 主包 ≤ 200KB（gzip 后），启用代码分割
- **B**: 主包 ≤ 500KB（gzip 后），基本代码分割
- **C**: 不限制（MVP 阶段不关注）

[Answer]: 后续提供的前端框架会有限制

---

### Q4: 可访问性（Accessibility）要求
前端是否需要满足可访问性标准？

- **A**: 满足 WCAG 2.1 AA 级别（推荐）
- **B**: 基本可访问性（键盘导航、语义化 HTML）
- **C**: 不做特殊要求（MVP 阶段）

[Answer]: A

---

### Q5: 响应式设计要求
前端是否需要支持移动端访问？

- **A**: 仅桌面端（内部系统，员工使用公司电脑）
- **B**: 响应式设计，支持平板和桌面
- **C**: 完全响应式，支持手机、平板、桌面

[Answer]: A

---

### Q6: 离线支持
是否需要支持离线访问或 PWA 功能？

- **A**: 不需要（始终在线使用）
- **B**: 基本离线提示（检测网络状态，显示友好提示）
- **C**: PWA 支持（Service Worker 缓存静态资源）

[Answer]: A

---

### Q7: 前端错误监控
是否需要集成前端错误监控？

- **A**: 不需要（MVP 阶段，依赖用户反馈）
- **B**: 基本错误捕获（ErrorBoundary + console.error）
- **C**: 集成第三方监控（如 Sentry）

[Answer]: A

---

### Q8: 图片优化策略
产品图片的优化策略是什么？

- **A**: 前端懒加载 + 占位图，后端不做处理
- **B**: 后端生成缩略图，前端按需加载不同尺寸
- **C**: 使用 CDN 图片服务（如 Cloudinary）

[Answer]: A

---

### Q9: API 请求超时配置
前端 API 请求的超时时间设置？

- **A**: 统一 10 秒超时
- **B**: 普通请求 10 秒，文件上传 60 秒
- **C**: 不设超时（依赖浏览器默认）

[Answer]: A

---

### Q10: 安全性 — XSS 防护
前端如何处理 XSS 防护？

- **A**: 依赖框架自动转义（React/Vue 默认行为）
- **B**: 框架转义 + 显式 sanitize 用户输入
- **C**: 框架转义 + CSP 策略

[Answer]: 后续提供的前端框架会有要求
