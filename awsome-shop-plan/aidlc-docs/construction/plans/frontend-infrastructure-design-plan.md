## 概述
本计划用于收集前端应用的基础设施配置决策，包括容器化部署、静态资源服务、环境变量等。

## 设计问题

### Q1: 前端部署方式
前端应用如何部署？

- **A**: Nginx 容器托管静态文件（推荐，与后端统一 Docker 部署）
- **B**: 独立 Node.js 服务（SSR 场景）
- **C**: 直接使用开发服务器（仅开发环境）

[Answer]: A

---

### Q2: 前端服务端口
前端 Nginx 容器对外暴露的端口？

- **A**: 80（标准 HTTP 端口，通过 Nginx 反向代理访问）
- **B**: 3000（开发习惯端口）
- **C**: 其他端口（请指定）

[Answer]: B

---

### Q3: API 代理配置
前端如何访问后端 API？

- **A**: Nginx 反向代理 /api/* 到 api-gateway:8080（推荐，避免跨域）
- **B**: 前端直接访问 api-gateway:8080（需处理 CORS）
- **C**: 通过环境变量配置 API 地址

[Answer]: A

---

### Q4: 静态资源缓存策略
静态资源（JS/CSS/图片）的缓存策略？

- **A**: 文件名 hash + 长期缓存（推荐，构建工具自动处理）
- **B**: 短期缓存（1 天）
- **C**: 不缓存（开发阶段）

[Answer]: C

---

### Q5: 健康检查配置
前端容器是否需要健康检查？

- **A**: 需要，检查 Nginx 是否正常响应（推荐）
- **B**: 不需要（前端为静态资源，Nginx 稳定性高）

[Answer]: A
