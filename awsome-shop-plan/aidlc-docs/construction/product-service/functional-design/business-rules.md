## 2. 业务规则

### BR-PROD-001: 产品软删除
- 删除产品时将 status 设为 INACTIVE，不物理删除数据
- INACTIVE 产品不在员工端展示
- 管理员可通过 status 筛选查看已下架产品
- 产品图片文件不随软删除而删除

### BR-PROD-002: 产品状态过滤
- 员工端 API（/api/products/*）仅返回 status = ACTIVE 的产品
- 管理员端 API（/api/admin/products/*）可查看所有状态的产品

### BR-PROD-003: 分类层级限制
- 最大层级深度为 2 级（一级分类 + 二级分类）
- parentId = NULL 为一级分类
- parentId 指向一级分类 ID 为二级分类
- 不允许 parentId 指向二级分类（即不允许三级分类）

### BR-PROD-004: 分类删除前置条件
- 删除分类前必须检查：无子分类 AND 无 ACTIVE 产品
- 有子分类时提示"请先删除子分类"
- 有 ACTIVE 产品时提示"请先移除该分类下的产品"

### BR-PROD-005: 分类筛选产品
- 按一级分类筛选时，返回该分类 + 所有二级子分类下的产品
- 按二级分类筛选时，仅返回该分类下的产品

### BR-PROD-006: 产品列表排序
- 默认按 created_at DESC 排序（最新上架在前）

### BR-PROD-007: 库存并发控制
- 库存扣减使用悲观锁（SELECT FOR UPDATE）
- 在同一事务中完成查询和更新
- 库存不足时返回错误，不执行扣减

### BR-PROD-008: 文件命名规则
- 上传文件使用 UUID + 原始扩展名重命名
- 避免文件名冲突和中文文件名问题

### BR-PROD-009: 产品名称允许重复
- 不同产品可以使用相同名称（与 US-013 验收标准一致）

## 3. 错误码定义

### 产品错误码

| 错误码 | HTTP 状态码 | 场景 |
|--------|-----------|------|
| PROD_001 | 404 | 分类不存在 |
| PROD_002 | 404 | 产品不存在 |
| PROD_003 | 409 | 库存不足 |
| PROD_004 | 400 | 参数校验失败 |

### 分类错误码

| 错误码 | HTTP 状态码 | 场景 |
|--------|-----------|------|
| CAT_001 | 404 | 分类不存在 |
| CAT_002 | 400 | 超过分类层级限制（最大2级） |
| CAT_003 | 409 | 分类下有子分类，无法删除 |
| CAT_004 | 409 | 分类下有产品，无法删除 |
| CAT_005 | 400 | 参数校验失败 |

### 文件错误码

| 错误码 | HTTP 状态码 | 场景 |
|--------|-----------|------|
| FILE_001 | 400 | 文件为空 |
| FILE_002 | 400 | 文件大小超过限制（5MB） |
| FILE_003 | 400 | 不支持的文件类型 |

## 4. 统一响应格式

与 auth-service 保持一致：

```json
// 成功响应
{
  "code": "SUCCESS",
  "message": "操作成功",
  "data": { ... }
}

// 错误响应
{
  "code": "PROD_003",
  "message": "库存不足",
  "data": null
}
```

## 5. 边界条件

| 场景 | 处理方式 |
|------|---------|
| 库存为 0 的产品 | 员工端正常展示，但前端应显示"已售罄" |
| 搜索关键词为空 | 返回所有 ACTIVE 产品 |
| 分类下无产品 | 返回空列表，前端显示"该分类暂无产品" |
| 上传同名文件 | UUID 重命名，不会覆盖 |
| 并发扣减库存 | 悲观锁保证串行，后到的请求可能因库存不足失败 |
| 删除已下架产品 | 已经是 INACTIVE 状态，返回成功（幂等） |
| 分页 size 超过 100 | 强制限制为 100 |
