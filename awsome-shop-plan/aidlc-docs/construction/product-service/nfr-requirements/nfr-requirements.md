# Unit 3: product-service — NFR 需求

---

## 安全性

### NFR-PROD-001: 输入校验
- 所有 API 输入参数必须进行校验
- 防止 SQL 注入（参数化查询）
- 防止 XSS（输出编码）
- 文件上传校验：类型白名单 + 大小限制

### NFR-PROD-002: 文件上传安全
- 仅允许图片类型：jpg, jpeg, png, gif, webp
- 文件大小上限：5MB
- 使用 UUID 重命名文件，防止路径遍历攻击
- 不信任客户端提供的 Content-Type，应根据文件内容或扩展名判断

### NFR-PROD-003: 内部接口安全
- 内部接口（/api/internal/*）仅通过 Docker 内部网络访问
- 不经过 API 网关，不暴露给外部
- 调用方通过 Docker DNS 直接访问

---

## 性能

### NFR-PROD-004: API 响应时间
- P95 响应时间目标：200ms 以内（常规查询接口）
- 文件上传接口因 I/O 操作，P95 目标放宽至 1000ms
- 产品列表查询利用数据库索引（category_id, name, status）

### NFR-PROD-005: 分页查询性能
- 产品列表和管理员列表支持分页，单页最大 100 条
- 分类筛选查询需高效：一级分类筛选时需查询子分类 ID 集合

### NFR-PROD-006: 库存扣减性能
- 悲观锁（SELECT FOR UPDATE）锁等待超时：5 秒
- 超时后快速失败，返回错误提示用户重试
- MVP 阶段并发量低，悲观锁性能影响可忽略

### NFR-PROD-007: 图片访问性能
- 图片文件通过 HTTP 直接返回文件流
- 设置 Cache-Control 响应头（public, max-age=86400）
- Nginx 层可进一步缓存静态资源

---

## 可靠性

### NFR-PROD-008: 数据一致性
- 库存扣减使用悲观锁保证不超卖
- 产品软删除保证历史数据完整性
- 分类删除前置条件检查（无子分类、无 ACTIVE 产品）

### NFR-PROD-009: 文件存储可靠性
- 图片文件存储在 Docker 卷挂载目录（宿主机持久化）
- MVP 阶段不做磁盘空间监控（产品数量有限）
- 软删除产品时不删除图片文件（可能被兑换历史引用）

---

## 可维护性

### NFR-PROD-010: 统一错误响应
- 所有错误使用统一格式：`{ code, message, data }`
- 错误码前缀：PROD_（产品）、CAT_（分类）、FILE_（文件）
- 业务异常返回 4xx，系统异常返回 5xx

### NFR-PROD-011: 日志规范
- 遵循后端统一技术框架的日志规范
- 关键操作记录日志：产品创建/更新/删除、分类变更、文件上传、库存扣减/恢复
- 库存扣减日志需包含：productId、quantity、扣减前库存、扣减后库存

### NFR-PROD-012: 配置外部化
- 所有配置通过环境变量注入
- 必需配置：DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD
- 可选配置：UPLOAD_DIR（默认 /app/uploads）、MAX_FILE_SIZE（默认 5MB）、SERVER_PORT

---

## 可测试性

### NFR-PROD-013: 单元测试
- 核心业务逻辑需有单元测试覆盖：产品 CRUD、分类层级校验、库存扣减
- 测试框架随后端技术框架确定

### NFR-PROD-014: API 测试
- 所有 API 端点需有集成测试覆盖
- 文件上传测试需使用测试图片文件
- 测试数据使用独立的测试数据库或内存数据库
