# Unit 1: awsomeshop-frontend — 业务逻辑模型

## 1. 认证流程

### 1.1 用户注册流程

```
用户访问 /register
  │
  ├── 填写注册表单
  │     ├── 用户名（必填）
  │     └── 密码（必填）
  │
  ├── 前端表单校验
  │     ├── 用户名：3-20位，字母数字下划线
  │     └── 密码：最少6位
  │
  ├── 提交注册请求
  │     └── POST /api/auth/register
  │
  ├── 处理响应
  │     ├── 成功 → 显示成功提示 → 跳转登录页
  │     └── 失败 → 显示错误提示（用户名已存在等）
  │
  └── 结束
```

### 1.2 用户登录流程

```
用户访问 /login
  │
  ├── 填写登录表单
  │     ├── 用户名（必填）
  │     └── 密码（必填）
  │
  ├── 提交登录请求
  │     └── POST /api/auth/login
  │
  ├── 处理响应
  │     ├── 成功
  │     │     ├── 存储 token 到 localStorage
  │     │     ├── 存储 user 到 localStorage
  │     │     ├── 更新认证状态
  │     │     └── 跳转首页（或原目标页）
  │     └── 失败 → 显示错误提示
  │
  └── 结束
```

### 1.3 用户退出流程

```
用户点击"退出登录"
  │
  ├── 清除 localStorage 中的 token 和 user
  │
  ├── 重置认证状态
  │
  ├── 跳转登录页
  │
  └── 结束
```

### 1.4 认证状态恢复流程

```
应用启动/页面刷新
  │
  ├── 从 localStorage 读取 token 和 user
  │
  ├── 判断 token 是否存在
  │     ├── 存在 → 恢复认证状态
  │     │         ├── 设置 isAuthenticated = true
  │     │         ├── 设置 user 信息
  │     │         └── 配置 HTTP 请求头 Authorization
  │     └── 不存在 → 保持未登录状态
  │
  └── 结束
```

## 2. 产品浏览流程

### 2.1 产品列表加载流程

```
用户访问 /products
  │
  ├── 初始化状态
  │     ├── page = 0
  │     ├── products = []
  │     └── hasMore = true
  │
  ├── 加载分类列表
  │     └── GET /api/categories/tree
  │
  ├── 加载首批产品
  │     └── GET /api/products?page=0&size=20
  │
  ├── 渲染产品卡片网格
  │
  ├── 用户滚动到底部（无限滚动）
  │     ├── 检查 hasMore
  │     │     ├── true → 加载下一页
  │     │     │         ├── page++
  │     │     │         ├── GET /api/products?page={page}&size=20
  │     │     │         ├── 追加到 products
  │     │     │         └── 更新 hasMore
  │     │     └── false → 显示"没有更多了"
  │
  └── 结束
```

### 2.2 分类筛选流程

```
用户点击分类
  │
  ├── 更新 selectedCategoryId
  │
  ├── 重置分页状态
  │     ├── page = 0
  │     ├── products = []
  │     └── hasMore = true
  │
  ├── 加载该分类下的产品
  │     └── GET /api/products?categoryId={id}&page=0&size=20
  │
  ├── 渲染产品卡片网格
  │
  └── 结束
```

### 2.3 产品搜索流程

```
用户输入搜索关键词
  │
  ├── 防抖处理（300ms）
  │
  ├── 更新 searchKeyword
  │
  ├── 重置分页状态
  │     ├── page = 0
  │     ├── products = []
  │     └── hasMore = true
  │
  ├── 搜索产品
  │     └── GET /api/products?keyword={keyword}&page=0&size=20
  │
  ├── 渲染搜索结果
  │
  └── 结束
```

### 2.4 产品详情查看流程

```
用户点击产品卡片
  │
  ├── 跳转 /products/:id
  │
  ├── 加载产品详情
  │     └── GET /api/products/:id
  │
  ├── 渲染产品详情
  │     ├── 产品图片（懒加载）
  │     ├── 产品名称
  │     ├── 产品描述
  │     ├── 所需积分
  │     ├── 库存数量
  │     └── 兑换按钮
  │
  └── 结束
```

## 3. 兑换流程

### 3.1 发起兑换流程

```
用户在产品详情页点击"兑换"
  │
  ├── 跳转兑换确认页 /orders/confirm/:productId
  │
  ├── 加载确认页数据
  │     ├── GET /api/products/:productId（产品信息）
  │     └── GET /api/points/balance（当前积分）
  │
  ├── 渲染确认页
  │     ├── 产品信息（图片、名称、描述）
  │     ├── 所需积分
  │     ├── 当前积分余额
  │     ├── 兑换后剩余积分
  │     └── 确认/取消按钮
  │
  ├── 用户点击"确认兑换"
  │     │
  │     ├── 前端校验
  │     │     └── 当前积分 >= 所需积分？
  │     │           ├── 否 → 显示"积分不足"提示
  │     │           └── 是 → 继续
  │     │
  │     ├── 提交兑换请求
  │     │     └── POST /api/orders { productId }
  │     │
  │     ├── 处理响应
  │     │     ├── 成功
  │     │     │     ├── 显示成功提示
  │     │     │     ├── 更新积分余额
  │     │     │     └── 跳转兑换历史页
  │     │     └── 失败
  │     │           ├── 积分不足 → 显示错误提示
  │     │           ├── 库存不足 → 显示错误提示
  │     │           └── 其他错误 → 显示错误提示
  │
  └── 结束
```

### 3.2 查看兑换历史流程

```
用户访问 /orders
  │
  ├── 初始化状态
  │     ├── page = 0

  │     └── 兑换时间
  │
  ├── 用户滚动到底部（无限滚动）

  │
  └── 结束
```

## 5. 管理端流程

### 5.1 产品管理流程

#### 产品列表
```
管理员访问 /admin/products
  │
  ├── 加载产品列表（传统分页）
  │     └── GET /api/admin/products?page=0&size=10
  │
  ├── 渲染产品表格
  │     ├── 产品图片
  │     ├── 产品名称
  │     ├── 分类
  │     ├── 积分价格
  │     ├── 库存
  │     ├── 状态
  │     └── 操作（编辑/删除）
  │
  ├── 分页导航
  │     └── 点击页码 → 加载对应页
  │
  └── 结束
```

#### 新增产品
```
管理员访问 /admin/products/new
  │
  ├── 加载分类列表
  │     └── GET /api/admin/categories
  │
  ├── 填写产品表单
  │     ├── 产品名称（必填）
  │     ├── 产品描述
  │     ├── 所需积分（必填，正整数）
  │     ├── 库存数量（必填，非负整数）
  │     ├── 产品分类（必填）
  │     └── 产品图片（可选，上传）
  │
  ├── 图片上传（如有）
  │     └── POST /api/files/upload
  │
  ├── 提交产品
  │     └── POST /api/admin/products
  │
  ├── 处理响应
  │     ├── 成功 → 显示成功提示 → 跳转产品列表
  │     └── 失败 → 显示错误提示
  │
  └── 结束
```

#### 编辑产品
```
管理员访问 /admin/products/:id/edit
  │
  ├── 加载产品详情
  │     └── GET /api/admin/products/:id
  │
  ├── 加载分类列表
  │     └── GET /api/admin/categories
  │
  ├── 填充表单
  │
  ├── 修改并提交
  │     └── PUT /api/admin/products/:id
  │
  ├── 处理响应
  │     ├── 成功 → 显示成功提示 → 跳转产品列表
  │     └── 失败 → 显示错误提示
  │
  └── 结束
```

#### 删除产品
```
管理员点击"删除"按钮
  │
  ├── 显示确认弹窗
  │
  ├── 确认删除
  │     └── DELETE /api/admin/products/:id
  │
  ├── 处理响应
  │     ├── 成功 → 显示成功提示 → 刷新列表
  │     └── 失败 → 显示错误提示（有关联订单等）
  │
  └── 结束
```

## 7. 错误处理流程

### 7.1 API 错误处理

```
API 请求返回错误
  │
  ├── 解析错误响应
  │     └── { code, message, data }
  │
  ├── 根据 HTTP 状态码处理
  │     ├── 401 → 清除认证状态 → 跳转登录页
  │     ├── 403 → 显示"权限不足"提示
  │     ├── 404 → 显示"资源不存在"提示
  │     ├── 500 → 显示"服务器错误"提示
  │     └── 其他 → 显示 message 内容
  │
  ├── 显示全局消息提示（Toast）
  │
  └── 结束
```

### 7.2 网络错误处理

```
网络请求失败
  │
  ├── 判断错误类型
  │     ├── 网络断开 → 显示"网络连接失败"
  │     ├── 请求超时 → 显示"请求超时，请重试"
  │     └── 其他 → 显示"请求失败"
  │
  ├── 显示全局消息提示
  │
  └── 结束
```
