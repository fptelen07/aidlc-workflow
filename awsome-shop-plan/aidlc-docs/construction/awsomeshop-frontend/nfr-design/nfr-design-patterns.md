## 1. 路由级代码分割

### 目的
减少首屏加载体积，提升首屏性能（NFR-FE-001）

### 设计模式
```
┌─────────────────────────────────────────────────────────────┐
│                        应用入口                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                    路由配置                          │   │
│  │  /login      → lazy(() => import('LoginPage'))      │   │
│  │  /products   → lazy(() => import('ProductListPage'))│   │
│  │  /admin/*    → lazy(() => import('AdminModule'))    │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  按需加载页面                        │   │
│  │  首次访问路由时才下载对应 chunk                      │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 实现要点
- 所有页面组件使用动态 import 懒加载
- 管理端模块整体打包为独立 chunk
- 公共依赖提取到 vendor chunk
- 加载中显示 Loading Spinner

### 覆盖 NFR
- NFR-FE-001: 首屏加载性能
- NFR-FE-003: 路由切换性能

## 2. 统一 HTTP 客户端封装

### 目的
统一 API 请求处理，实现超时、错误处理、认证（NFR-FE-014）

### 设计模式
```
┌─────────────────────────────────────────────────────────────┐
│                     HTTP 客户端封装                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   请求拦截器                         │   │
│  │  - 添加 Authorization: Bearer {token}               │   │
│  │  - 设置 Content-Type                                │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   超时配置                           │   │
│  │  - 默认超时: 10 秒                                   │   │
│  │  - 超时后抛出 TimeoutError                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   响应拦截器                         │   │
│  │  - 401: 清除 token，跳转登录页                      │   │
│  │  - 403: 显示无权限提示                              │   │
│  │  - 500: 显示服务器错误提示                          │   │
│  │  - 网络错误: 显示网络异常提示                       │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 伪代码
```typescript
// services/http.ts
const httpClient = createHttpClient({
  baseURL: '/api',
  timeout: 10000, // 10 秒
});

// 请求拦截器
httpClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器
httpClient.interceptors.response.use(
  (response) => response.data,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      localStorage.removeItem('user');
      router.push('/login');
    } else if (error.response?.status === 403) {
      showToast('无权限访问', 'error');
    } else if (error.response?.status >= 500) {
      showToast('服务器错误，请稍后重试', 'error');
    } else if (!error.response) {
      showToast('网络异常，请检查网络连接', 'error');
    }
    return Promise.reject(error);
  }
);
```

- NFR-FE-011: JWT 令牌存储
- NFR-FE-014: API 请求超时
- NFR-FE-016: 网络错误处理

## 3. 错误边界与降级

### 目的
防止组件错误导致整个应用崩溃（NFR-FE-015）

### 设计模式
```
┌─────────────────────────────────────────────────────────────┐
│                      错误边界层级                            │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              全局 ErrorBoundary                      │   │
│  │  捕获未处理的渲染错误，显示全局错误页面              │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │           页面级 ErrorBoundary               │   │   │
│  │  │  捕获页面组件错误，显示页面级错误提示        │   │   │
│  │  │  ┌─────────────────────────────────────┐   │   │   │
│  │  │  │      组件级 ErrorBoundary            │   │   │   │
│  │  │  │  捕获特定组件错误，显示占位组件      │   │   │   │
│  │  │  └─────────────────────────────────────┘   │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 降级策略
| 错误级别 | 降级方案 |
|---------|---------|
| 全局错误 | 显示"应用出错"页面，提供刷新按钮 |
| 页面错误 | 显示"页面加载失败"，提供返回/重试按钮 |
| 组件错误 | 显示占位组件，不影响其他组件 |

## 4. 图片懒加载与占位

### 目的
优化图片加载性能，提升用户体验（NFR-FE-020）

### 设计模式
```
┌─────────────────────────────────────────────────────────────┐
│                    LazyImage 组件                            │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                 初始状态                              │   │
│  │  显示占位图（灰色背景 + loading 动画）               │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │            Intersection Observer 监听                │   │
│  │  图片进入视口 → 开始加载真实图片                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────┐           ┌─────────────────────┐     │
│  │   加载成功       │           │     加载失败         │     │
│  │   显示真实图片   │           │   显示错误占位图     │     │
│  └─────────────────┘           └─────────────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### 伪代码
```typescript
// components/LazyImage.tsx
function LazyImage({ src, alt, ...props }) {
  const [loaded, setLoaded] = useState(false);
  const [error, setError] = useState(false);
  const imgRef = useRef(null);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          const img = new Image();
          img.src = src;
          img.onload = () => setLoaded(true);
          img.onerror = () => setError(true);
          observer.disconnect();
        }
      },
      { rootMargin: '100px' } // 提前 100px 开始加载
    );
    
    if (imgRef.current) {
      observer.observe(imgRef.current);
    }
    
    return () => observer.disconnect();
  }, [src]);

  if (error) {
    return <PlaceholderImage type="error" {...props} />;
  }
  
  if (!loaded) {
    return <PlaceholderImage type="loading" ref={imgRef} {...props} />;
  }
  
  return <img src={src} alt={alt} {...props} />;
}
```

#### 6.2 焦点管理
```typescript
// 模态框焦点陷阱
function Modal({ isOpen, onClose, children }) {
  const modalRef = useRef(null);
  
  useEffect(() => {
    if (isOpen) {
      // 保存之前的焦点
      const previousFocus = document.activeElement;
      // 聚焦到模态框
      modalRef.current?.focus();
      
      return () => {
        // 关闭时恢复焦点
        previousFocus?.focus();
      };
    }
  }, [isOpen]);
  
  // 焦点陷阱：Tab 只在模态框内循环
  const handleKeyDown = (e) => {
    if (e.key === 'Escape') {
      onClose();
    }
    if (e.key === 'Tab') {
      trapFocus(e, modalRef.current);
    }
  };
  
  return (
    <div
      ref={modalRef}
      role="dialog"
      aria-modal="true"
      tabIndex={-1}
      onKeyDown={handleKeyDown}
    >
      {children}
    </div>
  );
}
```
