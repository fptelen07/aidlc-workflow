# Unit 1: awsomeshop-frontend — NFR 需求

---

## 1. 性能需求

### NFR-FE-001: 首屏加载性能
- **需求**: 首屏内容绘制（FCP）P95 ≤ 2 秒
- **测量条件**: 4G 网络环境，桌面端 Chrome 浏览器
- **优先级**: 高

### NFR-FE-002: 页面交互响应
- **需求**: 用户交互（点击、输入）响应时间 ≤ 100ms
- **说明**: 不包含 API 请求等待时间
- **优先级**: 高

### NFR-FE-003: 路由切换性能
- **需求**: 页面路由切换时间 ≤ 300ms（不含数据加载）
- **说明**: 代码分割后的懒加载页面首次加载除外
- **优先级**: 中

### NFR-FE-004: 无限滚动性能
- **需求**: 产品列表无限滚动时，滚动帧率 ≥ 30fps
- **说明**: 避免滚动卡顿，使用虚拟列表或分批渲染
- **优先级**: 中

---

## 2. 可访问性需求

### NFR-FE-005: WCAG 2.1 AA 合规
- **需求**: 满足 WCAG 2.1 AA 级别可访问性标准
- **关键要求**:
  - 所有交互元素可通过键盘访问
  - 表单元素有关联的 label
  - 图片有 alt 属性
  - 颜色对比度符合标准（4.5:1 正文，3:1 大文本）
  - 焦点状态可见
- **优先级**: 高

### NFR-FE-006: 键盘导航
- **需求**: 所有功能可通过键盘完成操作
- **关键要求**:
  - Tab 键顺序合理
  - Enter/Space 可触发按钮
  - Escape 可关闭弹窗/下拉
  - 焦点陷阱用于模态框
- **优先级**: 高

### NFR-FE-007: 语义化 HTML
- **需求**: 使用语义化 HTML 标签
- **关键要求**:
  - 页面有唯一 h1 标题
  - 使用 nav、main、aside、footer 等语义标签
  - 列表使用 ul/ol/li
  - 表格使用 table/thead/tbody/th/td
- **优先级**: 高

### NFR-FE-008: 屏幕阅读器支持
- **需求**: 关键内容对屏幕阅读器友好
- **关键要求**:
  - 动态内容更新使用 aria-live
  - 图标按钮有 aria-label
  - 加载状态有 aria-busy
  - 错误提示关联到表单字段
- **优先级**: 中

---

## 3. 浏览器兼容性需求

### NFR-FE-009: 现代浏览器支持
- **需求**: 支持以下浏览器最新 2 个主版本
  - Google Chrome
  - Mozilla Firefox
  - Apple Safari
  - Microsoft Edge
- **说明**: 不支持 IE11
- **优先级**: 高

### NFR-FE-010: 桌面端优先
- **需求**: 仅支持桌面端访问
- **最小分辨率**: 1280 × 720
- **推荐分辨率**: 1920 × 1080
- **说明**: 内部系统，员工使用公司电脑访问
- **优先级**: 高

---

## 4. 安全性需求

### NFR-FE-011: JWT 令牌存储
- **需求**: JWT 令牌存储在 localStorage
- **安全措施**:
  - 退出登录时清除令牌
  - 令牌过期后自动跳转登录页
  - 不在 URL 中传递令牌
- **优先级**: 高

### NFR-FE-012: XSS 防护
- **需求**: 防止跨站脚本攻击
- **实现方式**: 依赖前端框架自动转义（具体框架在实现阶段确定）
- **关键要求**:
  - 不使用 innerHTML 直接插入用户内容
  - 富文本内容（如有）需 sanitize
- **优先级**: 高

### NFR-FE-013: 敏感信息保护
- **需求**: 不在前端存储或显示敏感信息
- **关键要求**:
  - 密码字段使用 type="password"
  - 不在 console 输出敏感数据
  - 不在 URL 参数中传递敏感信息
- **优先级**: 高

---

## 5. 可靠性需求

### NFR-FE-014: API 请求超时
- **需求**: 所有 API 请求设置 10 秒超时
- **超时处理**: 显示友好错误提示，允许用户重试
- **优先级**: 高

### NFR-FE-015: 错误边界
- **需求**: 组件级错误不导致整个应用崩溃
- **实现方式**: 使用 ErrorBoundary 捕获渲染错误
- **降级策略**: 显示错误提示组件，提供刷新按钮
- **优先级**: 高

### NFR-FE-016: 网络错误处理
- **需求**: 优雅处理网络错误
- **关键要求**:
  - 网络断开时显示友好提示
  - API 错误显示具体错误信息
  - 提供重试机制
- **优先级**: 高

---

## 6. 可维护性需求

### NFR-FE-017: 代码组织
- **需求**: 按功能模块组织代码
- **目录结构**:
  - components/ — 公共组件
  - pages/ — 页面组件
  - services/ — API 服务
  - stores/ — 状态管理
  - utils/ — 工具函数
  - types/ — 类型定义
- **优先级**: 中

### NFR-FE-018: 组件复用
- **需求**: 提取可复用组件
- **关键要求**:
  - 公共 UI 组件（按钮、输入框、表格等）统一封装
  - 业务组件按模块划分
  - 组件 props 有类型定义
- **优先级**: 中

### NFR-FE-019: 代码风格
- **需求**: 统一代码风格
- **实现方式**: 使用 ESLint + Prettier（具体配置在实现阶段确定）
- **优先级**: 中

---

## 7. 图片处理需求

### NFR-FE-020: 图片懒加载
- **需求**: 产品图片使用懒加载
- **实现方式**: 
  - 使用 Intersection Observer API
  - 图片进入视口前显示占位图
- **优先级**: 中

### NFR-FE-021: 图片加载失败处理
- **需求**: 图片加载失败时显示默认占位图
- **占位图**: 统一的"图片不可用"占位图
- **优先级**: 中

---

## 8. 用户体验需求

### NFR-FE-022: 加载状态反馈
- **需求**: 所有异步操作显示加载状态
- **关键要求**:
  - 页面加载显示 Loading Spinner
  - 按钮操作显示 loading 状态
  - 列表加载显示骨架屏或 loading 指示器
- **优先级**: 高

### NFR-FE-023: 操作反馈
- **需求**: 用户操作后给予明确反馈
- **关键要求**:
  - 成功操作显示成功提示（Toast）
  - 失败操作显示错误提示
  - 危险操作（删除）需二次确认
- **优先级**: 高

### NFR-FE-024: 空状态处理
- **需求**: 列表为空时显示友好的空状态
- **关键要求**:
  - 显示空状态插图和文案
  - 提供引导操作（如有）
- **优先级**: 中
