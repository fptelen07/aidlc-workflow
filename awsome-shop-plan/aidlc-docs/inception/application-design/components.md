# AWSomeShop 组件定义

## 系统架构概览

AWSomeShop 采用前后端分离架构，通过 RESTful API 通信，MySQL 持久化存储，Docker 容器化部署。

```
+------------------+     RESTful API     +------------------+     +-------+
|                  | ==================> |                  | ==> |       |
|   前端应用        |                     |   后端服务        |     | MySQL |
|   (Frontend)     | <================== |   (Backend)      | <== |       |
|                  |       JSON          |                  |     +-------+
+------------------+                     +------------------+
                                                |
                                                v
                                         +-------------+
                                         | 本地文件系统  |
                                         | (图片存储)   |
                                         +-------------+
```

## 前端组件

### FE-AUTH: 认证模块
- **职责**: 用户注册、登录、退出、令牌管理
- **接口**: 登录表单、注册表单、认证状态管理、路由守卫

### FE-PRODUCT: 产品模块
- **职责**: 产品列表展示、分类浏览、产品搜索、产品详情
- **接口**: 产品列表页、产品详情页、分类导航、搜索组件

### FE-POINTS: 积分模块
- **职责**: 积分余额展示、积分变动历史
- **接口**: 积分余额组件、积分历史列表页

### FE-ORDER: 兑换模块
- **职责**: 兑换操作、兑换确认、兑换历史
- **接口**: 兑换确认弹窗、兑换历史列表页、兑换详情页

### FE-ADMIN: 管理模块
- **职责**: 产品管理、分类管理、积分管理、兑换记录管理
- **接口**: 产品管理页、分类管理页、积分管理页、兑换记录页

### FE-COMMON: 公共模块
- **职责**: 布局、导航、分页、通知、HTTP 客户端封装
- **接口**: 主布局组件、导航栏、分页组件、消息提示组件

## 后端组件

### BE-AUTH: 认证组件
- **职责**: 用户注册、登录验证、JWT 令牌生成与校验、密码加密
- **接口**: RESTful API（/api/auth/*）

### BE-USER: 用户组件
- **职责**: 用户信息管理、角色管理
- **接口**: RESTful API（/api/users/*）

### BE-PRODUCT: 产品组件
- **职责**: 产品 CRUD、产品查询、库存管理
- **接口**: RESTful API（/api/products/*）

### BE-CATEGORY: 分类组件
- **职责**: 产品分类 CRUD、多级分类树管理
- **接口**: RESTful API（/api/categories/*）

### BE-POINTS: 积分组件
- **职责**: 积分余额管理、积分变动记录、积分自动发放（定时任务）
- **接口**: RESTful API（/api/points/*）

### BE-ORDER: 兑换组件
- **职责**: 兑换流程处理、兑换记录管理、兑换状态管理
- **接口**: RESTful API（/api/orders/*）

### BE-FILE: 文件组件
- **职责**: 产品图片上传、图片访问、本地文件系统管理
- **接口**: RESTful API（/api/files/*）

### BE-SCHEDULER: 调度组件
- **职责**: 定时任务管理、积分自动发放调度
- **接口**: 内部调度（无外部 API）

## 数据访问层

### DA-USER: 用户数据访问
- **职责**: 用户表 CRUD 操作
- **数据表**: users

### DA-PRODUCT: 产品数据访问
- **职责**: 产品表 CRUD 操作、库存更新
- **数据表**: products

### DA-CATEGORY: 分类数据访问
- **职责**: 分类表 CRUD 操作、分类树查询
- **数据表**: categories

### DA-POINTS: 积分数据访问
- **职责**: 积分余额读写、积分变动记录
- **数据表**: point_balances, point_transactions

### DA-ORDER: 兑换数据访问
- **职责**: 兑换记录 CRUD 操作
- **数据表**: orders

### DA-CONFIG: 配置数据访问
- **职责**: 系统配置读写（积分发放规则等）
- **数据表**: system_configs
