# AWSomeShop 用户画像

---

## 画像 1：技术型员工 — 李明

| 属性 | 描述 |
|------|------|
| **姓名** | 李明 |
| **角色** | 软件开发工程师 |
| **年龄** | 28 岁 |
| **技术水平** | 高 — 熟悉各类 Web 应用，习惯键盘操作和搜索 |

**目标**:
- 快速找到想要的产品并完成兑换
- 高效查看积分余额和兑换历史
- 希望有搜索和筛选功能，减少浏览时间

**痛点**:
- 不喜欢操作步骤繁琐的系统
- 对页面加载慢或响应迟钝零容忍
- 不愿意在不必要的确认弹窗上浪费时间

**使用场景**:
- 午休时间快速浏览新上架产品
- 月初查看积分到账情况
- 搜索特定品类的产品进行兑换

## 画像 2：非技术型员工 — 王芳

| 属性 | 描述 |
|------|------|
| **姓名** | 王芳 |
| **角色** | 行政助理 |
| **年龄** | 35 岁 |
| **技术水平** | 中低 — 日常使用办公软件，对复杂 Web 应用不太熟悉 |

**目标**:
- 轻松浏览产品，通过分类找到感兴趣的商品
- 清楚了解自己有多少积分、能兑换什么
- 操作简单直观，不需要学习成本

**痛点**:
- 界面复杂时容易迷失方向
- 不确定操作是否成功时会感到焦虑
- 不习惯使用搜索功能，更倾向于浏览分类

**使用场景**:
- 收到积分发放通知后登录查看余额
- 按分类逐级浏览产品
- 兑换后反复确认订单状态

## 画像 3：系统管理员 — 张伟

| 属性 | 描述 |
|------|------|
| **姓名** | 张伟 |
| **角色** | HR 部门系统管理员 |
| **年龄** | 32 岁 |
| **技术水平** | 中高 — 熟悉后台管理系统，有数据管理经验 |

**目标**:
- 高效管理产品目录（批量操作优于逐个操作）
- 准确管理员工积分，确保数据一致性
- 快速查看系统运营数据（兑换记录、积分使用情况）

**痛点**:
- 手动逐个操作大量数据时效率低下
- 操作失误后难以回溯和修正
- 缺乏数据概览，需要逐个查询

**使用场景**:
- 每月配置积分自动发放规则
- 新增或更新产品信息和分类
- 手动调整个别员工的积分（奖励或纠错）
- 查看兑换记录和积分使用统计

| 王芳（非技术型员工） | Employee | 分类浏览、积分查看、兑换操作 |
