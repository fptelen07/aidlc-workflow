# AWSomeShop 用户故事

## 旅程 1：员工注册与认证

### US-001: 员工注册账号
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要注册一个系统账号，以便我能够登录并使用积分兑换功能。

**验收标准**:
1. Given 员工访问注册页面，When 填写用户名、密码、姓名、工号并提交，Then 系统创建账号并提示注册成功
2. Given 员工填写注册信息，When 用户名已被占用，Then 系统提示"用户名已存在"并阻止注册
3. Given 员工填写注册信息，When 密码长度少于 8 位，Then 系统提示密码不符合要求
4. Given 员工填写注册信息，When 必填字段为空，Then 系统高亮显示缺失字段并提示填写
5. Given 注册成功，When 系统创建账号，Then 默认角色为"员工"，初始积分为 0

### US-002: 员工登录系统
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要使用用户名和密码登录系统，以便访问我的积分和产品页面。

**验收标准**:
1. Given 员工在登录页面，When 输入正确的用户名和密码并提交，Then 系统验证通过并跳转到首页
2. Given 员工在登录页面，When 输入错误的用户名或密码，Then 系统提示"用户名或密码错误"
3. Given 员工登录成功，When 系统生成认证令牌，Then 令牌包含用户角色信息
4. Given 员工已登录，When 令牌过期，Then 系统自动跳转到登录页面
5. Given 员工在登录页面，When 连续 5 次输入错误密码，Then 系统锁定账号 15 分钟

### US-003: 员工退出登录
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要安全退出系统，以便保护我的账号安全。

**验收标准**:
1. Given 员工已登录，When 点击退出按钮，Then 系统清除认证令牌并跳转到登录页面
2. Given 员工已退出，When 尝试访问受保护页面，Then 系统重定向到登录页面
3. Given 员工已退出，When 使用浏览器后退按钮，Then 不能访问之前的受保护页面
4. Given 员工点击退出，When 系统处理退出请求，Then 服务端令牌同时失效
5. Given 员工在多个标签页登录，When 在一个标签页退出，Then 其他标签页刷新后也需要重新登录

## 旅程 2：浏览与搜索产品

### US-004: 按分类浏览产品
**优先级**: Must Have  
**画像**: 王芳

> 作为一名员工，我想要按产品分类浏览商品，以便找到我感兴趣的品类。

**验收标准**:
1. Given 员工在产品页面，When 查看分类导航，Then 显示所有一级分类列表
2. Given 员工点击一级分类，When 该分类有子分类，Then 展开显示二级分类
3. Given 员工点击某个分类，When 该分类下有产品，Then 显示该分类及其子分类下的所有产品
4. Given 员工浏览分类产品，When 产品列表超过一页，Then 支持分页浏览
5. Given 员工点击某个分类，When 该分类下无产品，Then 显示"该分类暂无产品"提示
6. Given 员工在子分类页面，When 查看面包屑导航，Then 显示完整的分类路径（如：首页 > 电子产品 > 耳机）

### US-005: 搜索产品
**优先级**: Must Have  
**画像**: 李明

> 作为一名员工，我想要通过关键词搜索产品，以便快速找到我想要的商品。

**验收标准**:
1. Given 员工在产品页面，When 在搜索框输入产品名称关键词并提交，Then 显示匹配的产品列表
2. Given 员工搜索产品，When 搜索结果为空，Then 显示"未找到相关产品"提示
3. Given 员工搜索产品，When 输入部分关键词，Then 支持模糊匹配搜索
4. Given 搜索结果列表，When 结果超过一页，Then 支持分页浏览
5. Given 员工搜索产品，When 搜索框为空并提交，Then 显示所有产品列表

### US-006: 查看产品详情
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要查看产品的详细信息，以便决定是否兑换。

**验收标准**:
1. Given 员工在产品列表，When 点击某个产品，Then 显示产品详情页（名称、描述、图片、所需积分、库存）
2. Given 员工查看产品详情，When 产品有库存，Then 显示"兑换"按钮
3. Given 员工查看产品详情，When 产品无库存，Then 显示"暂无库存"并禁用兑换按钮
4. Given 员工查看产品详情，When 员工积分不足，Then 显示"积分不足"提示
5. Given 员工查看产品详情，When 查看所需积分，Then 同时显示员工当前积分余额便于对比

### US-007: 产品列表展示
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要在产品列表中看到关键信息，以便快速判断是否感兴趣。

**验收标准**:
1. Given 员工在产品列表页，When 查看产品卡片，Then 每个卡片显示产品图片、名称、所需积分
2. Given 员工在产品列表页，When 产品无库存，Then 产品卡片显示"已售罄"标识
3. Given 员工在产品列表页，When 产品列表加载，Then 页面加载时间不超过 3 秒
4. Given 员工在产品列表页，When 查看列表，Then 产品按上架时间倒序排列（最新在前）
5. Given 员工在产品列表页，When 列表为空，Then 显示友好的空状态提示

## 旅程 3：积分查看与管理

### US-008: 查看积分余额
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要查看我的当前积分余额，以便了解我可以兑换什么。

**验收标准**:
1. Given 员工已登录，When 进入首页或个人中心，Then 显示当前积分余额
2. Given 员工查看积分，When 积分刚发生变动，Then 显示最新的积分余额（实时更新）
3. Given 员工查看积分，When 页面加载，Then 积分余额在页面顶部或显眼位置展示
4. Given 员工查看积分，When 积分为 0，Then 正常显示"0"而非错误信息
5. Given 员工查看积分，When 积分数值较大，Then 使用千分位格式显示（如 1,500）

### US-009: 查看积分变动历史
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要查看积分变动历史，以便了解积分的来源和去向。

**验收标准**:
1. Given 员工在积分历史页面，When 查看列表，Then 显示每条记录的变动类型、数量、时间、备注
2. Given 员工查看积分历史，When 记录列表加载，Then 按时间倒序排列（最新在前）
3. Given 员工查看积分历史，When 变动类型为"发放"，Then 显示为正数（如 +100）
4. Given 员工查看积分历史，When 变动类型为"兑换"或"扣除"，Then 显示为负数（如 -50）
5. Given 员工查看积分历史，When 记录超过一页，Then 支持分页浏览
6. Given 员工查看积分历史，When 无任何记录，Then 显示"暂无积分变动记录"

## 旅程 4：兑换产品

### US-010: 发起产品兑换
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要使用积分兑换产品，以便获得我想要的商品。

**验收标准**:
1. Given 员工在产品详情页，When 点击"兑换"按钮，Then 显示兑换确认信息（产品名、所需积分、当前余额、兑换后余额）
2. Given 员工确认兑换，When 积分充足且有库存，Then 系统扣除积分、减少库存、创建兑换记录
3. Given 员工确认兑换，When 积分不足，Then 系统提示"积分不足，无法兑换"并阻止操作
4. Given 员工确认兑换，When 库存不足，Then 系统提示"库存不足"并阻止操作
5. Given 兑换成功，When 系统处理完成，Then 显示兑换成功页面，包含兑换单号和自取信息
6. Given 兑换过程中，When 发生并发兑换导致库存不足，Then 系统回滚积分扣除并提示用户

### US-011: 兑换确认
**优先级**: Must Have  
**画像**: 王芳

> 作为一名员工，我想要在兑换前看到确认信息，以便避免误操作。

**验收标准**:
1. Given 员工点击兑换，When 显示确认弹窗，Then 包含产品名称、所需积分、当前余额、兑换后余额
2. Given 确认弹窗显示，When 员工点击"确认兑换"，Then 执行兑换操作
3. Given 确认弹窗显示，When 员工点击"取消"，Then 关闭弹窗，不执行任何操作
4. Given 确认弹窗显示，When 员工查看信息，Then 兑换后余额自动计算并显示
5. Given 确认弹窗显示，When 兑换后余额为负数，Then 禁用"确认兑换"按钮

### US-012: 查看兑换历史
**优先级**: Must Have  
**画像**: 李明、王芳

> 作为一名员工，我想要查看我的兑换历史记录，以便跟踪我兑换过的产品。

**验收标准**:
1. Given 员工在兑换历史页面，When 查看列表，Then 显示每条记录的产品名称、消耗积分、兑换时间、状态
2. Given 员工查看兑换历史，When 记录列表加载，Then 按兑换时间倒序排列
3. Given 员工查看兑换历史，When 点击某条记录，Then 显示兑换详情（产品信息、兑换单号、自取信息）
4. Given 员工查看兑换历史，When 记录超过一页，Then 支持分页浏览
5. Given 员工查看兑换历史，When 无兑换记录，Then 显示"暂无兑换记录"提示
6. Given 员工查看兑换历史，When 查看状态列，Then 显示兑换状态（已兑换/已自取/已完成）

## 旅程 5：管理员 — 产品管理

### US-013: 添加新产品
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要添加新产品到系统中，以便员工可以浏览和兑换。

**验收标准**:
1. Given 管理员在产品管理页面，When 点击"添加产品"，Then 显示产品信息表单
2. Given 管理员填写产品信息，When 提交完整信息（名称、描述、积分、库存、分类、图片URL），Then 产品创建成功
3. Given 管理员填写产品信息，When 必填字段为空，Then 系统提示缺失字段
4. Given 管理员填写产品信息，When 积分或库存为负数，Then 系统提示输入无效
5. Given 产品创建成功，When 员工浏览产品列表，Then 新产品出现在对应分类中
6. Given 管理员添加产品，When 产品名称已存在，Then 系统允许创建（不同产品可同名）

### US-014: 编辑产品信息
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要编辑已有产品的信息，以便保持产品信息的准确性。

**验收标准**:
1. Given 管理员在产品列表，When 点击某产品的"编辑"按钮，Then 显示预填充的编辑表单
2. Given 管理员修改产品信息，When 提交修改，Then 产品信息更新成功
3. Given 管理员修改产品积分，When 有员工正在查看该产品，Then 员工刷新后看到最新积分
4. Given 管理员修改产品信息，When 修改后必填字段为空，Then 系统阻止提交
5. Given 管理员编辑产品，When 修改库存为 0，Then 产品在员工端显示"已售罄"

### US-015: 删除产品
**优先级**: Should Have  
**画像**: 张伟

> 作为管理员，我想要删除不再提供的产品，以便保持产品目录的整洁。

**验收标准**:
1. Given 管理员在产品列表，When 点击某产品的"删除"按钮，Then 显示删除确认弹窗
2. Given 删除确认弹窗，When 管理员确认删除，Then 产品从系统中移除
3. Given 删除确认弹窗，When 管理员取消，Then 不执行删除操作
4. Given 管理员删除产品，When 该产品有历史兑换记录，Then 兑换历史中仍保留该产品信息
5. Given 产品被删除，When 员工刷新产品列表，Then 该产品不再显示

### US-016: 查看产品列表（管理员视角）
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要查看所有产品的列表，以便管理产品目录。

**验收标准**:
1. Given 管理员在产品管理页面，When 查看产品列表，Then 显示所有产品（包含名称、分类、积分、库存、状态）
2. Given 管理员查看产品列表，When 产品库存为 0，Then 该产品行高亮显示"售罄"状态
3. Given 管理员查看产品列表，When 列表超过一页，Then 支持分页浏览
4. Given 管理员查看产品列表，When 搜索产品名称，Then 支持按名称筛选
5. Given 管理员查看产品列表，When 按分类筛选，Then 仅显示该分类下的产品

## 旅程 7：管理员 — 积分管理

**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要查看所有员工的积分余额，以便了解积分使用情况。

2. Given 管理员查看员工列表，When 列表超过一页，Then 支持分页浏览
3. Given 管理员查看员工列表，When 搜索员工姓名或工号，Then 支持按关键词筛选
4. Given 管理员查看员工列表，When 点击某员工，Then 显示该员工的积分变动明细
5. Given 管理员查看员工列表，When 列表加载，Then 按工号排序显示

### US-021: 手动调整员工积分
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要手动调整员工的积分，以便处理奖励发放或积分纠错。

**验收标准**:
1. Given 管理员在员工积分详情页，When 点击"调整积分"，Then 显示积分调整表单
2. Given 管理员填写调整信息，When 输入调整数量（正数为发放，负数为扣除）和备注，Then 积分调整成功
3. Given 管理员扣除积分，When 扣除后余额为负数，Then 系统提示"扣除后余额不足"并阻止操作
4. Given 积分调整成功，When 查看该员工积分历史，Then 显示本次调整记录（含操作人和备注）
5. Given 管理员调整积分，When 备注为空，Then 系统提示必须填写调整原因
6. Given 积分调整成功，When 员工登录查看，Then 看到最新的积分余额

### US-022: 配置积分自动发放规则
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要配置积分自动发放规则，以便系统按时自动为员工发放积分。

**验收标准**:
1. Given 管理员在积分配置页面，When 查看当前规则，Then 显示当前发放额度和发放周期
2. Given 管理员修改发放规则，When 设置发放额度和周期（如每月100积分），Then 规则保存成功
3. Given 管理员设置发放规则，When 额度为 0 或负数，Then 系统提示输入无效
4. Given 发放规则生效，When 到达发放周期，Then 系统自动为所有员工发放指定额度的积分
5. Given 自动发放完成，When 查看积分历史，Then 每位员工都有一条"系统自动发放"记录
6. Given 管理员修改规则，When 新规则保存，Then 下次发放按新规则执行

## 旅程 8：管理员 — 兑换记录管理

### US-023: 查看所有兑换记录
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要查看所有员工的兑换记录，以便监控兑换情况。

**验收标准**:
1. Given 管理员在兑换记录页面，When 查看列表，Then 显示所有兑换记录（员工姓名、产品名称、消耗积分、兑换时间、状态）
2. Given 管理员查看兑换记录，When 列表超过一页，Then 支持分页浏览
3. Given 管理员查看兑换记录，When 按员工姓名搜索，Then 筛选该员工的兑换记录
4. Given 管理员查看兑换记录，When 按时间范围筛选，Then 仅显示该时间段内的记录
5. Given 管理员查看兑换记录，When 列表加载，Then 按兑换时间倒序排列

### US-024: 更新兑换状态
**优先级**: Should Have  
**画像**: 张伟

> 作为管理员，我想要更新兑换记录的状态，以便跟踪员工是否已自取商品。

**验收标准**:
1. Given 管理员在兑换记录详情，When 点击"更新状态"，Then 显示状态选择（已兑换 → 已自取 → 已完成）
2. Given 管理员更新状态，When 选择新状态并确认，Then 状态更新成功
3. Given 状态更新成功，When 员工查看兑换历史，Then 看到最新的兑换状态
4. Given 管理员更新状态，When 状态只能前进不能后退（如不能从"已完成"改回"已兑换"），Then 系统限制非法状态变更
5. Given 管理员更新状态，When 操作完成，Then 记录状态变更时间和操作人

## 旅程 9：管理员 — 系统登录与权限

### US-025: 管理员登录
**优先级**: Must Have  
**画像**: 张伟

> 作为管理员，我想要登录系统并看到管理功能入口，以便执行管理操作。

**验收标准**:
1. Given 管理员在登录页面，When 输入管理员账号密码并登录，Then 系统识别管理员角色并显示管理功能菜单
2. Given 管理员登录成功，When 进入首页，Then 除了员工功能外，还显示管理入口（产品管理、积分管理、兑换记录、分类管理）
3. Given 管理员已登录，When 访问管理页面，Then 系统验证管理员权限后允许访问
4. Given 普通员工已登录，When 尝试访问管理页面 URL，Then 系统返回 403 禁止访问
5. Given 管理员已登录，When 同时也可以作为员工使用兑换功能，Then 管理员拥有员工的所有功能权限
