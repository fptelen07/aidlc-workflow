# 角色画像 (Personas)

## Persona 1: 员工 (Employee)

| 属性 | 描述 |
|------|------|
| **角色名称** | 李明（普通员工） |
| **角色类型** | employee |
| **目标** | 使用积分兑换心仪商品 |
| **动机** | 公司发放积分作为奖励，希望兑换实用商品 |
| **典型操作** | 浏览商品、按分类筛选、查看详情、兑换商品、查看兑换记录、查看积分余额和历史 |
| **技术水平** | 普通，期望简洁直观的操作界面 |
| **使用频率** | 每周 1-2 次浏览，每月 1-2 次兑换 |
| **痛点** | 不清楚积分余额、不知道兑换进度、找不到想要的商品 |

## Persona 2: 管理员 (Admin)

| 属性 | 描述 |
|------|------|
| **角色名称** | 王芳（商城管理员） |
| **角色类型** | admin |
| **目标** | 高效管理商城运营（商品、订单、积分、用户） |
| **动机** | 确保商城正常运转，及时处理兑换订单，合理分配积分 |
| **典型操作** | 管理商品上下架、处理兑换订单、发放积分、查看统计数据、管理用户 |
| **技术水平** | 中等，熟悉后台管理系统操作 |
| **使用频率** | 每天使用，处理订单和日常管理 |
| **痛点** | 批量操作效率低、缺乏数据统计、订单处理流程繁琐 |
